ply

A ratio-based CSS framework built for AI agents, accessible by default, and small enough to ship anywhere.

One stylesheet. ~18KB gzipped. Dark mode, responsive grid, semantic HTML — no build step, no JavaScript, no configuration.

Why ply?

CSS frameworks were designed for humans reading documentation. But increasingly, the first consumer of your framework is an AI — a coding agent in Replit, Claude, Cursor, or Copilot generating a UI from a prompt. ply is designed for that world.

AI-Native

ply ships with PLY.md — a complete class reference that any AI agent can read. It also includes ply-classes.json for programmatic tool use, and copy-paste snippets that agents emit directly.

Class names are predictable. .alert-blue, .btn-sm, .unit-50 — an LLM guesses correctly on the first try. No memorizing utility shorthands or looking up obscure class prefixes.

Accessible by Default

Accessibility isn't a feature you bolt on — it's a civil rights issue. When your UI isn't accessible, you're excluding people. ply makes the right thing the easy thing:

  • Visible :focus-visible outlines on all interactive elements for keyboard navigation
  • .sr-only class for screen-reader-only content
  • prefers-reduced-motion respected globally — animations are disabled automatically
  • prefers-color-scheme dark mode — no flash, no toggle required
  • Semantic HTML styled out of the box — <details>, <dialog>, <progress>, <nav>, <table>
  • Color contrast meets WCAG AA across light and dark themes

When an AI generates ply markup, the output is accessible without the developer needing to think about it.

Small Footprint

The full framework is ~18KB gzipped. The core bundle is ~16KB. The essentials (grid + helpers only) is under 5KB.

No JavaScript runtime. No build step. No tree-shaking required. One <link> tag and you're done. That matters when you're spinning up a prototype in Replit or shipping to users on slow connections.

Built for Replit

ply is designed to be the CSS framework you reach for when an AI agent is building your UI in Replit.

Zero Config

ply includes .replit and replit.nix configuration files. Fork the repo, open in Replit, and start building. No npm install, no webpack, no PostCSS pipeline — the compiled CSS is already in dist/.

AI Agent Friendly

When Replit's AI agent needs to generate a dashboard, a form, or a landing page, it reads PLY.md and gets the complete class reference, usage rules, and ready-to-use snippets. The output is correct, accessible, and styled — on the first generation.

How ply Does Accessibility

Most frameworks treat accessibility as a docs page you're supposed to read. ply bakes it into the CSS itself so that correct behavior is the default, not the exception.

Keyboard Navigation

Every interactive element — links, buttons, inputs, tabs — gets a visible :focus-visible outline. Keyboard users always know where they are. Try it: press Tab through this page.

Reduced Motion

Users who set prefers-reduced-motion: reduce in their OS get all animations and transitions disabled automatically. No opt-in required, no extra classes.

This matters for people with vestibular disorders, motion sensitivity, or seizure conditions. The CSS handles it globally:

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

Screen Reader Support

The .sr-only class hides content visually while keeping it accessible to screen readers. Use it for labels, descriptions, and skip links:

<a href="#main" class="sr-only">Skip to content</a>

Semantic HTML, Styled

ply styles native HTML elements so you don't need wrapper divs or extra classes. When you use the right element, you get the right styling:

Native <details> — no JavaScript needed

This is a styled disclosure widget. It works with keyboard, screen readers, and has a smooth visual indicator. Zero classes added.

Get Started

Add one line to your HTML:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/plygrid@1/dist/css/ply.min.css">

That's it. Start writing semantic HTML and ply handles the rest.
